# Image Generation Runtime And Browser Coverage Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add one narrow runtime session helper plus two minimal browser specs so image generation is covered at the component-side orchestration boundary, the real page interaction boundary, and the real browser route-contract boundary, without replacing the existing `tsx` tests.

**Architecture:** `src/lib/image-studio-session.ts` becomes the new runtime seam: it receives already-prepared generation inputs from `ImageStudio`, delegates network work to `callImageStudioProxy()`, and delegates request sequencing to `executeImageStudioRequestStrategy()`. `src/components/image-studio.tsx` keeps prompt construction, React state, abort/timeout ownership, progress updates, and toast behavior, while Playwright runs two browser specs against the real Next dev server: one mocked `/api/images` smoke spec for DOM behavior and one real `/api/images` route-contract spec backed by a local mock upstream server. Chromium runs the full browser matrix; Firefox runs only the stable happy-path plus complete-failure subset.

**Tech Stack:** Next.js 16 App Router, React 19 client components, TypeScript, Node `assert`, `tsx`, Playwright, pnpm, ESLint

## Finalized Implementation Notes

- Keep `src/lib/image-request.ts`, `src/lib/image-studio-generation.ts`, and `src/lib/image-studio-proxy.ts` behaviorally unchanged. Keep `src/app/api/images/route.ts` unchanged unless the real browser route-contract spec reveals a verified production bug. The new session helper composes the existing proxy and request-strategy helpers instead of replacing them.
- Let `src/lib/image-studio-session.ts` derive `hasInputImages` from `options.images.length > 0` so `ImageStudio` only passes prepared files, not duplicate strategy state.
- Return `endpoint`, `debug`, `quality`, `qualityReported`, `size`, `sizeReported`, `images`, `firstError`, and `isPartial` from the session helper so `ImageStudio` can preserve its current success, partial, timeout, and abort UI behavior with minimal edits.
- Keep browser coverage English-only by setting Playwright locale to `en-US`; the spec explicitly excludes multi-language browser assertions.
- Avoid adding test-only production selectors unless the existing accessible labels and summary text prove insufficient. The first pass should use `#prompt`, `#api-key`, `input[type="file"]`, result image alt text, and summary label text that already exist today.
- Keep `tests/browser/image-studio-smoke.spec.ts` intercepting `/api/images` directly so those cases stay DOM-focused and deterministic.
- Add `tests/browser/image-studio-route-contract.spec.ts` that points the app at a local mock upstream server and lets the real `/api/images` route select `/v1/images/generations` versus `/v1/images/edits`.
- In the route-contract spec, assert the upstream request path, `n`, `prompt`, uploaded image file part(s), and any timeout-related multipart field only if the current route actually forwards one.
- Extend the mocked browser smoke spec with four unhappy-path cases: complete failure, partial success with a later failure, stop while pending, and timeout while pending.
- Keep that unhappy-path extension browser-only. Do not add new route/helper/runtime coverage for it unless browser coverage reveals a real production bug.
- Prefer test-only edits for the browser expansion. Production changes are allowed only to fix verified route-contract or browser interaction bugs.
- Chromium runs the full browser suite.
- Firefox runs only the stable subset: mocked happy paths, mocked complete failure, and the route-contract happy paths.
- Keep partial-success, stop, and timeout browser coverage Chromium-only for now.
- Keep browser assertions minimal and behavior-oriented: request-count stability, visible failure/stopped/timed-out feedback, retained partial images, and the absence of unexpected result images.
- The complete-failure smoke should intercept one `/api/images` request, show failure feedback, and leave the result gallery empty.
- The partial-success smoke should set image count above `1`, keep the earlier image(s) visible, show the partial warning, and prove the final request count stops growing once retries finish.
- The stop smoke should hold `/api/images` pending, click `Stop`, show `Generation stopped.`, and prove no extra `/api/images` requests begin after the stop.
- The timeout smoke should set `#request-timeout` to `5`, hold `/api/images` pending past the deadline, show `Generation timed out after 5s.`, and keep the result gallery empty.
- `tests/image-studio-stop-timeout-controls.test.ts` is the only pre-existing test file that should need source updates, because it currently pins the inline component-to-proxy call that the new session seam intentionally removes.

```ts
import { type Locale } from "@/lib/i18n"
import { executeImageStudioRequestStrategy } from "@/lib/image-studio-generation"
import { type GeneratedImage } from "@/lib/image-request"
import { callImageStudioProxy, type ImageStudioProxyResult } from "@/lib/image-studio-proxy"

export type RunImageStudioSessionOptions<TDebug = unknown> = {
  apiKey: string
  background: string
  endpoint: string
  imageCount: number
  images: readonly File[]
  isControlError?: (error: unknown) => boolean
  locale: Locale
  model: string
  onImagesUpdated?: (images: GeneratedImage[]) => void
  onProxyResult?: (result: ImageStudioProxyResult<TDebug>) => void
  outputFormat: string
  prompt: string
  quality: string
  signal: AbortSignal
  size: string
  timeoutMs: number
}

export type ImageStudioSessionResult<TDebug = unknown> = {
  debug: TDebug | null
  endpoint: string
  firstError: unknown
  images: GeneratedImage[]
  isPartial: boolean
  quality: string
  qualityReported: boolean
  size: string
  sizeReported: boolean
}

export async function runImageStudioSession<TDebug = unknown>(
  options: RunImageStudioSessionOptions<TDebug>
): Promise<ImageStudioSessionResult<TDebug>> {
  let latestDebug: TDebug | null = null
  let latestEndpoint = options.endpoint.trim()
  let latestQuality = options.quality
  let latestQualityReported = false
  let latestSize = options.size
  let latestSizeReported = false

  const requestResult = await executeImageStudioRequestStrategy({
    total: options.imageCount,
    hasInputImages: options.images.length > 0,
    request: async (requestedCount) => {
      const proxyResult = await callImageStudioProxy<TDebug>({
        apiKey: options.apiKey,
        background: options.background,
        endpoint: options.endpoint,
        imageCount: requestedCount,
        images: options.images,
        locale: options.locale,
        model: options.model,
        outputFormat: options.outputFormat,
        prompt: options.prompt,
        quality: options.quality,
        signal: options.signal,
        size: options.size,
        timeoutMs: options.timeoutMs,
      })

      latestDebug = proxyResult.debug || latestDebug
      latestEndpoint = proxyResult.endpoint
      latestQuality = proxyResult.quality || latestQuality
      latestQualityReported = proxyResult.qualityReported
      latestSize = proxyResult.size || latestSize
      latestSizeReported = proxyResult.sizeReported
      options.onProxyResult?.(proxyResult)

      return proxyResult
    },
    selectImages: (proxyResult) => proxyResult.images,
    onImagesUpdated: options.onImagesUpdated,
    isControlError: options.isControlError,
  })

  return {
    debug: latestDebug,
    endpoint: latestEndpoint,
    firstError: requestResult.firstError,
    images: requestResult.images,
    isPartial: requestResult.isPartial,
    quality: latestQuality,
    qualityReported: latestQualityReported,
    size: latestSize,
    sizeReported: latestSizeReported,
  }
}
```

```ts
function parseMultipartRequest(request: Request): MultipartSnapshot {
  const contentType = request.headers()["content-type"] || ""
  const boundaryMatch = /boundary=([^;]+)/i.exec(contentType)

  if (!boundaryMatch) {
    throw new Error(`missing multipart boundary: ${contentType}`)
  }

  const body = request.postDataBuffer()

  if (!body) {
    throw new Error("missing multipart request body")
  }

  const boundary = `--${boundaryMatch[1]}`
  const parts = body.toString("latin1").split(boundary).slice(1, -1)
  const fields: Record<string, string[]> = {}
  const files: MultipartSnapshot["files"] = []

  for (const part of parts) {
    const normalizedPart = part.replace(/^\r\n/, "").replace(/\r\n$/, "")
    const separatorIndex = normalizedPart.indexOf("\r\n\r\n")

    if (separatorIndex === -1) {
      continue
    }

    const rawHeaders = normalizedPart.slice(0, separatorIndex)
    const rawValue = normalizedPart.slice(separatorIndex + 4).replace(/\r\n$/, "")
    const dispositionMatch = /name="([^"]+)"(?:; filename="([^"]+)")?/i.exec(rawHeaders)

    if (!dispositionMatch) {
      continue
    }

    const fieldName = dispositionMatch[1]
    const filename = dispositionMatch[2]

    if (filename) {
      const fileContentType = /content-type:\s*([^\r\n]+)/i.exec(rawHeaders)?.[1] || ""
      files.push({ contentType: fileContentType, fieldName, filename })
      continue
    }

    fields[fieldName] = [...(fields[fieldName] || []), rawValue]
  }

  return { fields, files }
}

  const quietPeriodMs = 350

    const currentCount = intercepted.length

      lastCountChangeAt = now

    return currentCount === count && now - lastCountChangeAt >= quietPeriodMs
  }, {
    message: `expected /api/images request count to remain at ${count} after the UI settled`,
    timeout: 1_000,
    intervals: [50, 100, 200],
  }).toBe(true)
}
```

```ts
function parseRequestSnapshot(path: string, contentType: string, body: Buffer): RequestSnapshot {
  if (contentType.includes("application/json")) {
    const payload = JSON.parse(body.toString("utf8")) as Record<string, unknown>
    const fields = Object.fromEntries(
      Object.entries(payload).map(([key, value]) => [key, Array.isArray(value) ? value.map(String) : [String(value)]])
    )

    return { fields, files: [], path }
  }

  const { fields, files } = parseMultipartBody(contentType, body)
  return { fields, files, path }
}

async function startMockUpstreamServer() {
  const requests: RequestSnapshot[] = []
  const server = createServer(async (request, response) => {
    const chunks: Buffer[] = []

    for await (const chunk of request) {
      chunks.push(Buffer.isBuffer(chunk) ? chunk : Buffer.from(chunk))
    }

    const pathname = new URL(request.url || "/", "http://127.0.0.1").pathname
    const contentType = Array.isArray(request.headers["content-type"])
      ? request.headers["content-type"][0] || ""
      : request.headers["content-type"] || ""

    requests.push(parseRequestSnapshot(pathname, contentType, Buffer.concat(chunks)))

    response.writeHead(200, { "content-type": "application/json" })
    response.end(
      JSON.stringify({
        created: 1,
        data: [{ b64_json: PNG_BASE64, revised_prompt: pathname.endsWith("/edits") ? "route-edit" : "route-generate" }],
      })
    )
  })

  server.listen(0, "127.0.0.1")
  await once(server, "listening")

  const address = server.address()

  if (!address || typeof address === "string") {
    throw new Error("mock upstream server did not expose a TCP port")
  }

  return {
    baseUrl: `http://127.0.0.1:${address.port}/v1`,
    close: () => new Promise<void>((resolve, reject) => {
      server.close((error) => error ? reject(error) : resolve())
    }),
    requests,
  }
}
```
